Contribute to EchoPoint

We welcome contributions to the development of EchoPoint through either of the following means:

  • Contribute to the development and porting of components. There are a lot of components that need to be ported over, as well as potentially new components that reflect the state of the art in AJAX toolkits. The more people we get who are interested in creating custom components, the better EchoPoint will become.
  • Donate to the project. Donations help us allocate more time and resources to the development work. Please use the paypal button to make a donation to the project. You can use the message field to request the area or component for which you would like your donation to apply. You will be listed as a project sponsor in the Sponsors section.

 

EchoPoint Component Library

EchoPoint component library for NextApp Echo3 web application framework. This project is a re-implementation of the EchoPointNG library created by Brad Baker for NextApp Echo2 web application framework.

  • The Component List shows the list of components that will be re-implemented in EchoPoint.
  • Interested developers read the Developer Guide for notes on getting involved with the re-implementation work.
  • View the test application for the client-side framework. This will be updated periodically as the test application is enhanced with tests for new components added to the library. You may also download the client and server test applications that are available under Documents & files>releases.
    • Client test application. Note that you can unarchive the contents and test off your filesystem without having to deploy to an application server.
    • Server test application. This will need to be deployed to an application server to work.
  • View the server-side component API specifications.

History

EchoPoint was created by Brad Baker as a rich component library for Echo1. The project was later re-implemented as EchoPointNG for Echo2 by Brad. Brad has stopped work on EchoPoint due to time and family constraints.

This project has been started by Sans Pareil Technologies, Inc., who wishes to continue to use the extensive component library developed by Brad. No further work on EchoPointNG is planned, however a new component library that replicates most of the common components of EchoPointNG for Echo3 framework will be developed.

Echo has changed quite radically from version 2 to 3 in that most component rendering is done in pure JavaScript on the client-side, thus reducing the server-side resource utilisation of Echo applications. A regular port of EchoPointNG is hence not possible, and requires total rewrite of the rendering peers for all components.

Source Code

The sources for the re-implementation work as well as the original EchoPointNG may be checked out from the project subversion repository.
  • EchoPoint subversion repository.
    • Access the repository trunk at https://echopoint.dev.java.net/svn/echopoint/trunk.
    • View repository trunk.
    • Note that it is not recommended that users use the trunk area since the status of the sources are likely to be unstable and not entirely tested. The trunk area is primarily for the use of developers involved with implementing the library. It is recommended that users use the latest version from the tags/current area.
      • Access tags area at https://echopoint.dev.java.net/svn/echopoint/tags/current.
      • View tags area. Check out the latest tagged release after viewing available tagged releases and use in your projects.
  • EchoPointNG subversion repository.
    • Access repository at https://echopoint.dev.java.net/svn/echopoint/branches/ng.
    • View repository.

Dependencies

The following libraries are used to build EchoPoint. You will need them in your applications to satisfy runtime dependencies. The release ZIP files contain the necessary libraries to build and deploy your applications using EchoPoint.
  • Echo3, Echo3Extras
  • XStream 1.3+
  • Apache Commons File Upload and IO packages
  • mime-uitl
The following additional libraries are used to test and build the library. These libraries are not included in the release package.
  • JUnit 4.4 - Used in the server-side test application.
  • yuicompressor 2.3.5 - Used to package the client-side test application.
